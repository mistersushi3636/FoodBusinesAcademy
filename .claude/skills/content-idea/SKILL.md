# content-idea

Генератор идей контента под 5 пилларов Food Business Academy. Учитывает уже существующие идеи и боли ЦА.

## Когда вызывать

- «придумай 10 идей»
- «идеи на следующую неделю»
- «что снимать про <тему>»
- «нужны идеи для шортсов»

## Алгоритм

1. **Прочитай контекст:**
   - `content/pillars.md` — 5 рубрик и триггерные слова
   - `brand/audience.md` — портреты и боли
   - `brand/tone-of-voice.md` — какие темы Антон НЕ берёт
2. **Список существующих идей:** просканируй `content/ideas/` (.md файлы) и собери уже использованные темы — НЕ дублируй.
3. **Уточни у пользователя** (если не указано):
   - Сколько идей нужно (default: 10)
   - Под какой формат: пост / шортс / длинное видео / любой
   - Фокус на конкретный пиллар или микс
4. **Сгенерируй** с равномерным распределением по пилларам (или по запросу):

Для каждой идеи:
- **Хук** (заголовок-обещание)
- **Пиллар** (💰/⚙️/📣/📖/🔮)
- **Портрет ЦА** (под кого)
- **Формат** (пост / шортс / видео)
- **Краткий тезис** (что внутри, 1–2 предложения)
- **Личная зацепка** (что Антон может рассказать из опыта — иначе идея слабая)

5. **Сохрани топ-идеи** (по запросу) каждую в отдельный файл `content/ideas/YYYY-MM-DD-<slug>.md` со структурой:

```markdown
---
created: YYYY-MM-DD
pillar: <emoji>
audience: <portrait>
format: <post|short|video>
status: idea  # idea | in_work | done
---

# <Хук>

**Тезис:** <что внутри>
**Личная зацепка:** <чем Антон отвечает на это из опыта>
**Рабочие материалы:** <ссылки на транскрипты, кейсы, заметки если есть>
```

## Правило качества

Каждая идея должна пройти 4 проверки из `content/pillars.md`:
- ✅ Попадает в один из 5 пилларов
- ✅ Закрывает боль одного из 3 портретов
- ✅ Есть личный опыт/кейс/цифра у Антона
- ✅ Можно рассказать в формате «я делал → получилось так»

Если хоть один ❌ — идею отбрасывай.

## Не выдумывай

Если предлагаешь тему, по которой у Антона может не быть опыта — пометь её 🚧 и спроси: «Антон, есть у тебя кейс по этой теме?»
