## 3. Полный pipeline-flow (от идеи до публикации)

```

    ↓
```

## 4. Где Антон, где агенты — точки касания

### Где Антон работает РУКАМИ (минимум):

| Задача | Частота | Время |
|---|---|---|
| Вбить метрики прошлой недели | 1×/нед (ПН до 10:30) | ~10-15 мин |
| Подтвердить план / дать правки | 1×/нед (ПН ~12:00) | ~5-10 мин |
| Записать голосовое / идею / кейс | по желанию | 2-5 мин |
| Опубликовать готовое в TG/TikTok/IG | по слотам | ~5 мин/пост |
| Снять видео / сделать фото для b-roll | 1-2×/нед | по задаче |
| Сгенерировать картинки в DALL-E | по постам | 2-3 мин/картинка |
| Финальная сверка перед публикацией | по постам | 1-2 мин |

**Итого Антон тратит:** ~3-5 часов/неделя на стратегию + публикации (без съёмки).

### Где работают агенты (автоматика):

- Парсинг конкурентов (раз в 7 дней)
- Анализ метрик
- Стратегический план 30 дней
- A/B варианты
- Структурирование текстов под платформы
- Генерация ТЗ дизайнеру
- Self-critique
- Канбан и календарь синхронизация

## 5. Анализ системы — плюсы и минусы

### ✅ Что работает хорошо

1. **Один источник правды** — Obsidian vault, всё в markdown, версионируется
2. **Автоматизированный еженедельный цикл** — cron + content-planner закрывают рутину
3. **A/B тестирование на уровне процесса** — гипотезы проверяются системно
4. **Метрики → обучение** — analytics корректирует другие агенты
5. **Архив невышедшего** — ничего не пропадает
6. **Уточняющие вопросы вшиты** — агенты не угадывают
7. **Прецеденты в JOURNAL.md** — система учится решать спорные ситуации
8. **Голос Антона централизован** — brand/tone-of-voice читается всеми агентами

### ⚠️ Узкие места (что неудобно Антону)

1. **🔴 Метрики вручную** — раз в неделю Антон вбивает цифры из 5+ платформ. Скучно. Высокий риск пропустить.
2. **🔴 Картинки руками** — агент даёт DALL-E промпт, но Антон сам копирует в OpenAI и генерирует.
3. **🔴 Голосовые → расшифровка вручную** — нет автотранскрипции в pipeline.
4. **🟡 Публикация вручную** — Антон сам копирует пост из vault в TG/TikTok.
5. **🟡 Нет мобильного интерфейса** — Obsidian неудобен с телефона для быстрого ответа на план.
6. **🟡 Cron живёт только в сессии Claude Code** — каждый рестарт надо пересоздавать.
7. **🟡 Нет уведомлений** — если cron сработал и план готов, Антон может не заметить.

## 6. План улучшений (10 шагов с приоритетами)

### Приоритет 🔥 (внедрить в первую очередь)

**1. Telegram-бот для Антона как личный хаб (1-2 дня работы)**
- Голосовое в бота → авто-транскрипция → orchestrator
- Уведомления когда план готов
- Inline-кнопки для ответа «ок / правки»
- Ввод метрик через диалог («сколько views у поста X?»)

**Зачем:** убирает 3 узких места разом — голосовые, метрики, уведомления. Антон вообще может не открывать Obsidian в обычной неделе.

---

**2. Persistent cron через GitHub Actions или /schedule (1 день)**
- Текущий cron живёт только в сессии Claude Code → восстанавливать каждый рестарт
- Альтернатива: запушить vault на GitHub → GitHub Actions запускает /content-planner по расписанию

**Зачем:** надёжность. Cron не пропадёт даже если Claude Code не запущен.

---

**3. Whisper API для расшифровки голосовых (полдня)**
- MCP с OpenAI → Whisper → автотранскрипция в Markdown
- Триггер: файл в `inbox/voice/` → агент расшифровывает → orchestrator получает текст

**Зачем:** Антон работает голосом, не текстом. Это 5x скорость генерации сырья.

---

### Приоритет ✅ (вторая волна)

**4. DALL-E API через MCP (полдня)**
- Designer-brief генерирует промпт → автогенерация картинки → сохранение в `assets/`
- Антон видит готовую картинку, а не текстовый промпт

**Зачем:** убирает шаг копи-паста. Картинки готовы сразу.

---

**5. Метрики через парсинг публичных счётчиков (1 день)**
- Tweepy / Telethon / TikTok API для базовых метрик (views, likes)
- Антон не вбивает, агент сам собирает

**Зачем:** Антон не забывает вбить метрики, цикл работает на автомате.

---

**6. Obsidian Calendar plugin + Dataview (30 минут)**
- Визуальный календарь публикаций прямо в Obsidian
- Цвета по пилларам, статусам

**Зачем:** Антон видит расклад на месяц одним взглядом.

---

**7. Bootstrap research дописать (после rate-limit reset)**
- 4 файла: content-plan-frameworks, viral-hooks-library, restaurant-marketing-cases, engagement-patterns

**Зачем:** content-planner будет работать с базой знаний, а не с пустыми руками.

---

### Приоритет 🟡 (когда будет время)

**8. Telegram автопостинг через бота (1 день)**
- Бот публикует за Антона по расписанию
- Антон только подтверждает «опубликовать сейчас?»

---

**9. Notion sync (опционально, 2-3 часа)**

## 7. Рекомендуемые MCP-сервисы

> MCP (Model Context Protocol) — расширения, которые дают Claude Code доступ к внешним сервисам.

### 🔥 Установить сразу

| MCP | Что даёт | Польза для FBA |
|---|---|---|

## 10. Резюме одной фразой

**Сейчас:** автоматизирован еженедельный цикл от метрик до плана с A/B вариантами, всё в Obsidian, агенты задают уточняющие вопросы и эскалируют через orchestrator.

**После топ-3 улучшений:** Антон работает голосом со смартфона через Telegram-бот, картинки и расшифровки делаются автоматически, cron не пропадает.

**Главная цель архитектуры:** Антон тратит время только на то, что только он может делать — записать голосовое, выбрать вариант, поставить лайк или дать правку. Всё остальное делают агенты.
