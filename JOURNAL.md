## 2026-04-21 — Архитектура субагентов: Variant B + bank по формату (C)

**Решение:** построена полноценная команда из 18 субагентов по варианту B (платформенные + функциональные) с банком контента по варианту C (группировка по формату).

**Структура:**
- `agents/orchestrator.md` — главный координатор
- `agents/functional/` — 6 общих агентов (content-adapter, research, calendar, analytics, designer, editor)
- `agents/platform/` — 8 платформенных (Telegram как HUB; TikTok, Instagram, YouTube — приоритет; VK, Dzen, Threads — вторичные; MAX — зеркало)
- `agents/lead-magnets/` — 3 специалиста воронки (researcher → architect → producer)
- `content-bank/` — ядра по форматам (short-video, long-video, text, visual)
- `platforms/<name>/` — готовые адаптированные материалы
- `design-system/` — единый бренд + промпты Nano Banana 2 и Kling 3.0
- `DASHBOARD.md` — переработан в минималистичный командный центр

**Почему:**
- Telegram — главный хаб, остальные платформы ведут туда через CTA
- Один контент → много платформенных версий (экономия сил Антона)
- `analytics` обучает других агентов на основе реальных метрик (цикл обратной связи)
- Единый визуальный стиль через `designer` + AI-инструменты

**Последствия:**
- Каждая платформа — отдельный подпроект с индивидуальным планом
- При появлении сырья от Антона — orchestrator раздаёт работу специалистам параллельно
- Метрики собираются централизованно и корректируют инструкции агентов
- Весь vault — источник правды, синхронизация через Obsidian

## 2026-04-20 — Obsidian как рабочее пространство
