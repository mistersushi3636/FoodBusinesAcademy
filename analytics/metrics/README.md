# 📊 Metrics — еженедельные метрики

## Как пользоваться

Каждый понедельник утром до 11:00 (до запуска content-planner) Антон заполняет файл `YYYY-WW.md` за прошедшую неделю.

Если не заполнить → content-marketer эскалирует: «Нет метрик за последнюю неделю».

---

## Формат файла: `YYYY-WW.md`

Скопируй шаблон ниже в новый файл с номером недели (ISO):
- 2026-W17 = неделя с 20 по 26 апреля 2026

```yaml
---
week: 2026-W17
period: "2026-04-20 .. 2026-04-26"
status: complete | pending
---
```

## Правила заполнения

1. **Заполнять честно** — даже если пост провалился, записать цифры
2. **Не округлять сильно** — 1247 → писать 1247, не 1200
3. **Пустые ячейки допустимы** — если метрика недоступна, оставить пустой
4. **Дедлайн** — понедельник 10:30 (за 30 мин до запуска content-planner)
5. **Заметки важны** — инсайт в конце файла часто ценнее цифр

---

## Связь с content-marketer

content-planner skill читает последний файл в этой папке при каждом запуске (понедельник 11:00).

На основе метрик:
- Классифицирует посты (🔥 топ / 🟡 средние / ❌ провал)
- Определяет какие ХУКИ работают
- Ребалансирует пиллары
- Корректирует план на следующие 30 дней
