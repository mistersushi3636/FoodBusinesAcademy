## 🎁 1. Lead-magnet-bot

**Что делает:** новый пользователь пишет `/start` → бот отдаёт PDF-чек-лист «10 ошибок при запуске доставки» в обмен на подписку на канал.

**Сценарий:**
1. `/start` → приветствие + кнопка «Получить чек-лист»
2. Просьба подписаться на канал (проверка через `getChatMember`)
3. После проверки — отправка PDF + краткое представление Антона
4. Через 2 дня — авто-фолоап с предложением подписаться на YouTube
5. Через 7 дней — приглашение на бесплатную консультацию (если запустим)

**Стек:**
- Python 3.11+
- aiogram 3.x
- SQLite для хранения подписчиков (для аналитики и фолоапов)
- APScheduler для отложенных сообщений
- Хостинг: VPS (Reg.ru / Selectel) или free-tier (Deta / Railway)

**Метрики:**
- /start → подписка на канал (conversion rate)
- Доля прошедших фолоап
- Стоимость подписчика, если есть платный трафик

**Бюджет разработки:** ~10–15 часов первой версии.

## 🤖 2. Content-assistant (для Антона)

**Что делает:** Антон присылает голосовое — бот возвращает готовый пост, идеи для шортсов и тезисы для длинного видео.

**Сценарий:**
1. Антон отправляет голосовое (5–60 мин)
2. Бот: транскрибирует через Whisper (OpenAI API) или локально
3. Сохраняет расшифровку в `knowledge-base/transcripts/YYYY-MM-DD.md`
4. Прогоняет через Claude API с промптом из `voice-to-post` skill
5. Возвращает: пост в TG (готовый), 3 идеи шортсов, тезисы длинного видео

**Стек:**
- Python + aiogram
- OpenAI Whisper API (или whisper.cpp локально для экономии)
- Claude API (Sonnet 4.6 для качества, Haiku 4.5 для черновиков)
- Prompt caching для tone-of-voice (фиксированные инструкции в кеше)

**Кому доступен:** только Антону (whitelist по telegram user_id).

**Бюджет разработки:** ~15–20 часов первой версии.

## 🧠 3. AI-консультант для аудитории

**Что делает:** аудитория задаёт боту вопросы по ресторанному бизнесу → бот отвечает голосом Антона на основе его материалов (RAG).

**Сценарий:**
1. Пользователь пишет вопрос
2. Бот ищет релевантные куски в `knowledge-base/` (vector search)
3. Claude API формирует ответ в тоне Антона + ссылки на оригинальные материалы (видео/посты)
4. Если уверенности нет — «не уверен, лучше задать Антону на консультации»
5. Лимит: N вопросов в день для бесплатных, безлимит для платных подписчиков

**Стек:**
- Python + aiogram
- Векторная БД: Qdrant (open-source, есть управляемое облако) или Chroma локально
- Embeddings: OpenAI `text-embedding-3-small` или локальные (multilingual-e5)
- Claude API для генерации ответа + prompt caching
- База контента: `knowledge-base/` + расшифровки видео + посты канала

**Этап запуска:** когда накопится 30+ материалов в knowledge-base (примерно через 2 месяца).

**Бюджет разработки:** ~40–60 часов первой версии.

## 💳 4. Sales-bot

**Что делает:** принимает заявки на консультации/курсы, ведёт по воронке, обрабатывает оплату.

**Сценарий:**
1. Пользователь приходит из канала по ссылке `/start consult`
2. Бот рассказывает форматы (1ч консультация / 4ч аудит / месячное наставничество)
3. Анкета: о бизнесе, выручка, точка боли
4. Бронирование слота через Calendly / TG-mini-app
5. Оплата через ЮKassa / Telegram Stars
6. После оплаты — материалы для подготовки + ссылка на встречу

**Стек:**
- Python + aiogram
- ЮKassa API (юр.лицо нужно)
- Calendly API или своя календарь-логика
- Notion API для CRM лидов (опц.)

**Этап запуска:** когда будет хотя бы 1 продукт + первые желающие купить (месяц 2–3).

**Бюджет разработки:** ~20–30 часов первой версии.

## Общие принципы

- **Один Python-проект, монорепо ботов** — общие утилиты, общий `.env`
- **Все боты отвечают команде `/help`** одинаковым форматом
- **Логирование:** все события → SQLite + еженедельная сводка
- **Безопасность:** все секреты в `.env`, никогда в коде
- **Бэкапы:** SQLite дампы раз в день в облако (S3 / Yandex Object Storage)
