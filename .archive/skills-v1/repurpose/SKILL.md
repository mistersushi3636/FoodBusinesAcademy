# repurpose

Один источник → 5–7 единиц контента на разные площадки. Главный рычаг продуктивности при 5–10 ч/неделю.

## Когда вызывать

- «переупакуй это видео»
- «нарежь шортсов и постов из расшифровки»
- «сделай контент на неделю из этого»

## Алгоритм

1. **Прочитай контекст:**
   - `brand/tone-of-voice.md`
   - `content/pillars.md`
   - `platforms/telegram-strategy.md` + `platforms/shorts-strategy.md`
2. **Прочитай источник** (расшифровка из `knowledge-base/transcripts/...` или указанный файл)
3. **Определи** главный пиллар и портрет ЦА
4. **Выдели мини-темы:** в источнике обычно 5–10 разных тезисов / историй / цифр. Список их.
5. **Раскидай по форматам:**

### Что должно получиться (минимум)

- **1 длинное видео YouTube** — главная тема целиком. Используй скилл `script-writer` (long).
- **3–5 шортсов** — каждый на одну острую мысль / цифру / провал. Используй `script-writer` (short).
- **2–3 поста в Telegram** — разбор, кейс, инструмент. Используй `voice-to-post` логику.
- **1 длинная статья** — для `content/articles/` и кросс-публикации в Дзен/VK (опц.).

### Структура вывода

Создай папку `content/calendar/YYYY-WW-<тема>/` (если ещё не существует), внутри:

```
content/calendar/2026-WW-<slug>/
├── README.md           # сводка: тема недели, источник, что получилось
├── youtube-long.md     # сценарий длинного видео
├── short-1.md          # шортс 1
├── short-2.md          # шортс 2
├── short-3.md          # шортс 3
├── tg-post-1.md        # пост 1 (разбор)
├── tg-post-2.md        # пост 2 (история)
├── tg-post-3.md        # пост 3 (короткая мысль)
└── article.md          # длинная статья (опц.)
```

В `README.md`:

```markdown
# Неделя WW — Тема: <название>

**Источник:** `knowledge-base/transcripts/<file>.md`
**Пиллар:** <emoji>
**Портрет ЦА:** <portrait>
**Дата создания:** YYYY-MM-DD

## Что в папке
| Файл | Формат | Хук | Статус |
|---|---|---|---|
| youtube-long.md | YouTube 12 мин | <хук> | draft |
| short-1.md | Shorts | <хук> | draft |
| ...
```

## Правила

- **Не повторяй один тезис в двух форматах одинаково.** Каждый формат — другая упаковка одной мысли.
- **Шортсы** должны иметь разные хуки (шок-цифра, провал, контр-интуитив).
- **Посты** должны попадать в разные подзадачи (один — инструмент, второй — история, третий — мысль).
- **Каждая единица контента** проходит проверку 4 пунктов из `pillars.md`.
- Если в источнике мало материала — честно скажи: «вытянем 3 единицы вместо 7, источник лёгкий».
