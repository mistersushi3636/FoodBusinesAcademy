# content-adapter

Превращает чужой экспертный контент (канал, видео, статью, подкаст) в **оригинальные материалы** для Food Business Academy. Суть и мысль сохраняются, формулировки и примеры полностью переписываются под ресторанный бизнес и опыт Антона.

## Когда вызывать

- «разбери канал @xyz»
- «адаптируй этот пост/видео под нас»
- «переложи идеи Гребенюка/Аяза/автора на ресторанку»
- «вытащи полезное из этого источника»

---

## Принципы адаптации (обязательно)

**Что сохраняем:**
- ядро идеи (тезис, инсайт, вывод)
- структуру рассуждения (шаги, этапы, фреймворк)
- логическую рамку (причина → следствие)

**Что меняем полностью:**
- все фразы и формулировки (антиплагиат — перепиши с нуля своими словами)
- все примеры и истории → на ресторанные из опыта «Мистер Суши»
- все цифры → Антоновы (фудкост, средний чек, комиссии агрегаторов, смены поваров)
- метафоры и образы → из кухни/зала/доставки
- лексика → `brand/tone-of-voice.md` (на «ты», без бизнес-сленга без расшифровки)

**Категорически нельзя:**
- копировать дословно даже одну фразу длиннее 4–5 слов подряд
- использовать фирменные термины автора (если у Гребенюка «отдел продаж на максималках» — не повторять)
- упоминать автора источника в публикациях (вариант A — скрытая адаптация)
- выдумывать цифры и кейсы Антона, если их нет

**Проверка на плагиат после адаптации:** если в готовом тексте нашёл 5+ подряд одинаковых слов с источником — переписать.

## Алгоритм (6 шагов)

### Шаг 1. Сбор материала

Варианты входа:
- **URL Telegram-канала** (`@name` или `t.me/name`) → `WebFetch` на `https://t.me/s/<name>` (публичное превью) для последних постов + `WebSearch` для био автора и интервью.
- **URL YouTube** → `WebFetch` описания/комментов + попросить у пользователя транскрипт, если глубокий разбор.
- **Статья/пост** (URL) → `WebFetch`.
- **Файл в проекте** (транскрипт, PDF, скриншот) → читай напрямую.
- **Картинки/схемы от пользователя** → анализируй визуально (я мультимодальный).

**Если доступ ограничен:** честно сказать «вижу только последние N постов, для глубокого разбора нужен экспорт из Telegram Desktop → HTML/JSON архив».

### Шаг 2. Профиль источника

Сделай паспорт источника — 1 страница:

```markdown
# Паспорт источника: <имя>

**Кто автор:** короткое био (сфера, статус, годы опыта).
**Про что канал/источник:** 1–2 предложения.
**Основная аудитория источника:** кто их смотрит.
**Ключевые фреймворки/методы:** список 3–7 (коротко).
**Фирменные термины/метафоры:** что не берём дословно.
**Частотные темы:** топ-5 о чём говорит.
**Tone of voice:** как говорит (эмоция, лексика).
**Что МОЖЕТ быть полезно для ресторанки:** первичный фильтр.
```

### Шаг 3. Фильтр по релевантности

Для каждой идеи из источника проверь:

1. ✅ Ложится ли на один из 5 пилларов Food Business Academy? (`content/pillars.md`)
2. ✅ Закрывает ли боль хотя бы одного из 3 портретов ЦА? (`brand/audience.md`)
3. ✅ Есть ли у Антона опыт в этой теме? (если нет — пометь 🚧 «нужен кейс от Антона»)
4. ✅ Можно ли переложить без потери смысла в контекст ресторана/кухни/доставки?

Если хоть один ❌ — выбрасывай идею из короткого списка. Лучше 3 сильные, чем 10 средних.

### Шаг 4. Адаптация (ядро скилла)

Для каждой отобранной идеи сделай **карточку адаптации**:

```markdown
## Идея <N>: <короткое название>

**Источник (внутренне):** <ссылка/пост-ID/время видео>
**Ядро (что сохраняем):** <тезис в 1 предложение>
**Фреймворк (если есть):** <шаги/структура>

### Перевод на ресторанку
- Автор говорил про: <сфера источника>
- У Антона это выглядит как: <ресторанный эквивалент>
- Цифра/кейс от Антона: <конкретика из Мистер Суши>

### Антиплагиат-чек
- Не использую фразы: <список характерных выражений автора>
- Новые формулировки: <как это скажет Антон>
```

### Шаг 5. Генерация материалов (по дефолту — средний пакет)

Из 3–5 сильных адаптированных идей собери:

**Дефолтный пакет за прогон:**
- 3 поста в Telegram (разные форматы: разбор, история, короткая мысль)
- 2 сценария шортсов (60 сек)
- 5–7 записей в банк идей (`content/ideas/`)

**По запросу:**
- `быстро` → только идеи в банк
- `полный пакет` → + 1 статья + 1 сценарий YouTube

Каждый материал создавай через уже существующие шаблоны `_templates/`:
- Пост → `_templates/post.md`
- Идея → `_templates/idea.md`
- Сценарий → `_templates/script.md`

Каждый материал обязан пройти проверку 4 пунктов из `pillars.md` и `tone-of-voice.md`.

### Шаг 6. Блок «Визуал» — в каждом материале

К каждому готовому материалу добавляй секцию `## Визуал`:

```markdown
## Визуал

**Концепция:** <что показываем, главная мысль визуала в 1 строке>
**Форма:** схема / инфографика / карусель / обложка / B-roll-описание / мем

### Mermaid-диаграмма (если подходит)
```mermaid
<диаграмма>
```

### Промпт для AI-генератора (Midjourney / DALL-E / Flux / Nano Banana)
<детальный промпт на английском — стиль, композиция, свет, цвет>

### ТЗ для ручной отрисовки (Canva/Figma)
- Формат: 1080×1350 (пост) / 1080×1920 (шортс-обложка) / 1080×1080 (квадрат)
- Текст на картинке: <точные слова, 3–5 строк максимум>
- Визуальные элементы: <что изобразить — иконки, фигуры, графики>
- Палитра: <цвета бренда, если заданы, иначе — тёплые ресторанные>
- Шрифт: <категория — sans-serif жирный заголовок>

### Карусель (если формат — карусель)
- Слайд 1 (хук): <текст + визуал>
- Слайд 2: ...
- Слайд N (CTA): ...
```

**Правила для визуала:**
- Если у источника была схема/презентация и она приложена — **полностью перерисовать**: поменять расположение, иконки, цвета, подписи. Суть — как у Антона, форма — новая.
- Не копировать композицию слайдов автора.
- Для схем предпочтительно Mermaid (рендерится в Obsidian и площадках), для «живых» картинок — AI-промпт.
- Тексты на визуале — короткие, в тоне Антона.

## Структура вывода (куда всё сохраняется)

Создай папку `knowledge-base/external-sources/<slug-источника>/`:

```
knowledge-base/external-sources/grebenyuk-telegram/
├── README.md                  # паспорт источника + сводка разбора
├── raw/                       # сырые выгрузки (посты, скриншоты, транскрипты)
│   └── posts-2026-04-20.md
├── adaptations.md             # все карточки адаптации из шага 4
├── output/                    # готовые материалы из шага 5
│   ├── tg-post-1.md
│   ├── tg-post-2.md
│   ├── tg-post-3.md
│   ├── short-1.md
│   ├── short-2.md
│   └── ideas/                 # копии в content/ideas/ (основные), здесь только список
└── plagiarism-check.md        # чек, какие фразы из источника НЕ использовать
```

Одновременно идеи (шаг 5, банк) сохрани и в `content/ideas/YYYY-MM-DD-<slug>.md` (чтобы попали в Dataview на MOC).

**Что выброшено и почему:** 2–3 строки
**Что создано:**
- 3 поста → <ссылки>
- 2 шортса → <ссылки>
- 5 идей в банк → <ссылки>
**Нужны кейсы от Антона для:** список тем с 🚧

## Не выдумывай

Если у Антона может не быть опыта по теме — пометь 🚧 и **не придумывай** цифры/истории Мистер Суши. Лучше оставь идею в банке с пометкой «нужна голосовуха от Антона на эту тему».

## Ограничения технические

- Telegram-каналы доступны через `t.me/s/<name>` (публичное превью, ~20–50 последних постов). Для полного разбора — пользователь экспортирует канал через Telegram Desktop и кладёт архив в `raw/`.
- YouTube транскрипты — через внешние сервисы или пользовательский экспорт; без транскрипта работай только с описанием и комментами.
- Если источник платный/закрытый — честно сказать «нет доступа, нужен экспорт от тебя».
