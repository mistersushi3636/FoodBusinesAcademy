# 🤖 Agents — Команда субагентов Food Business Academy

> Каждый субагент — это специалист со своей зоной ответственности. Антон создаёт сырьё → Orchestrator распределяет → специалисты адаптируют и доводят до результата.

---

## 🎯 Командная структура

```
        Anton (создаёт сырьё: голосовые, идеи, кейсы, метрики)
                    ↓
           🎯 Orchestrator (распределяет работу)
                    ↓
        📊 content-marketer (стратегия 30 дней, понедельник 11:00)
                    ↓
              calendar (раскладка по датам)
                    ↓
    ┌───────────────┼────────────────┬──────────────┐
    ↓               ↓                ↓              ↓
⚙️ Functional   📱 Platform     💎 Lead Magnets   🎨 Design
(обслуживают    (адаптация       (воронка         (единый
 всех)          под платформу)   подписчика)       стиль)
                                                        ↑
              📊 analytics (метрики → обратная связь)──┘
```

**Главная цель всех платформенных агентов:** вести аудиторию в Telegram-канал (основной хаб).

**Еженедельный цикл (понедельник 11:00):** cron → content-marketer → план → calendar → platform-агенты → designer → editor → публикация → метрики Антона → analytics → обратно в content-marketer.

## 🔄 Как это работает на практике

1. **Антон** записывает голосовое / кидает идею / присылает кейс в чат с Claude
2. **Orchestrator** определяет: это новый материал, разбор источника или реакция на тренд
3. **Functional-агенты** делают общую работу (адаптация, дизайн, календарь)
4. **Platform-агенты** готовят версии под свои площадки (все с CTA в Telegram)
5. **Analytics** отслеживает результат → корректирует инструкции других агентов
6. **Всё синхронизируется в Obsidian** (этот vault — источник правды)

## 📏 Правила взаимодействия (ОБЯЗАТЕЛЬНО для всех агентов)

### 1. Уточняющие вопросы — обязательны

**Когда агент ПОЛУЧАЕТ задачу:**
- Если непонятен любой параметр (цель, дедлайн, формат, аудитория, A/B вариант) → ОБЯЗАН задать уточняющий вопрос
- Не угадывать. Не домысливать. Не выполнять по умолчанию
- Возвращать вопрос постановщику задачи

**Когда агент СТАВИТ задачу:**
- Давать полный контекст: тема + цель + дедлайн + формат + связанные материалы
- Если получает уточнение — отвечать максимально конкретно
- Не отмахиваться формулировкой «сам разберись»

### 2. Эскалация вверх

Если агент не может принять решение сам (этический вопрос / конфликт приоритетов / большой ребаланс / нет данных):
→ Эскалирует через orchestrator → Антон принимает решение

### 3. Иерархия задач

```
Антон → orchestrator → content-marketer (стратегия) → calendar (даты) → 
  → platform-агент (адаптация) → content-adapter / designer / editor → публикация
```

Каждый агент уважает иерархию: не обходит вверх (не идёт сам к Антону, идёт через orchestrator), не перепрыгивает горизонтально (telegram не делает работу tiktok).

### 4. Источник правды и стиль

- Единый источник правды — **этот vault** (все материалы в `content-bank/` и `platforms/`)
- CTA → **Telegram** на всех платформах, кроме самого TG
- Единый визуал — через [[design-system/README|design-system]] (DALL-E/GPT Image, Canva, Kling 3.0)
- Голос бренда — [[brand/tone-of-voice|Tone of Voice]]
- Каждый агент обязан обновлять метрики после публикации (Антон вбивает в `analytics/metrics/YYYY-WW.md` вручную)

### 5. A/B варианты (для контент-цикла)

Каждый материал на ближайшую неделю должен иметь 2 варианта с разными хуками (один эмо, один экспертный). Антон выбирает один → второй идёт в архив (`content/kanban.md` → колонка «Архив»).
