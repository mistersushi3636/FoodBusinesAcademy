# 3 скрипта администратора при жалобе гостя

**Тезис:** администратор не импровизирует, когда гость недоволен. Есть проверенная логика: слушай → признай → реши → закрепи.

**Личная зацепка:** 🚧 какие 3 типовые жалобы в Мистер Суши? Примеры:
1. Блюдо холодное / неправильное
2. Долго ждали / доставка на час позже
3. Волос/ошибка в блюде

**Формат:** практический инструмент (набор скриптов).

**Что внутри:**
- Типовая структура ответа (слушай → признай ошибку → предложи решение → компенсация → спасибо)
- 3 готовых скрипта (холодное блюдо, долгая доставка, брак)
- Что НЕ говорить (оправдания вместо решения)
- Как документировать для аналитики
